# CLAUDE.md - Trade Manager Project

## プロジェクト概要

ゴールドスキャルピングの収支管理ダッシュボード。
MT4（XM口座）のトレードデータを自動取得して可視化するWebアプリ。

---

## 技術スタック

- **フロントエンド**：React + Vite + TypeScript + TailwindCSS + shadcn/ui + Recharts
- **バックエンド**：Python + FastAPI + SQLAlchemy + Pydantic
- **データベース**：PostgreSQL（Supabase）
- **MT4連携**：MQL4のEA → FastAPI WebhookでHTTP送信
- **バージョン管理**：Git + GitHub

## UIデザイン方針

- レイアウト：折りたたみサイドバー（アイコン表示） + マルチパネルのプラットフォーム風
- サイドバーは通常時アイコンのみ表示、展開するとページ名が見える
- メインエリアは複数パネルに分割してトレーディングツール風に見せる
- shadcn/ui の Sidebar コンポーネントを活用する
- カラーテーマ：Slate Professional（落ち着いたグレートーン、アクセント色は薄紫 #8b8cf8）

## コーディング規則

- Pythonのフォーマットはblackを使用
- フロントエンドは TypeScript（`.ts` / `.tsx`）で記述する
- Reactは関数コンポーネントで統一
- 変数名・関数名は英語で記述
- コメントは日本語でOK
- 環境変数は必ず`.env`で管理、コードへのハードコード禁止
- APIキー・DB接続情報は絶対にコードに書かない・GitHubに上げない

## 学習方針（最重要）

このプロジェクトのユーザーはプログラミング・開発が完全未経験。
以下の方針を必ず守ること。

### 教え方について

- コードを一方的に生成して渡すのではなく、**何をしているか・なぜそう書くかを説明してから**手順を示す
- ファイルの作成・コードの記述は**基本的にユーザー自身が行う**。Claude が代わりにやるのではなく、「このファイルを作って、この内容を書いて」という形で指示する
- 専門用語を使う場合は必ずわかりやすい言葉で補足説明する
- 「なんとなくわかる」ではなく「ちゃんとわかった」と言えるレベルまで丁寧に説明する

### エラー対応について

- エラーが出たときは**答えを直接出さない**
- 「このエラーはこういう意味で、原因はここにある可能性が高い。まずここを確認してみて」という形で考え方を教える
- ユーザーが自分で原因を特定・修正できるように誘導する

### コードレビューについて

- コードを書いたら「これは何をしているコードか」を自分の言葉で説明してもらう
- 理解が曖昧な箇所は先に進まず、その場で解消する

### ゴール

- AIが書いたコードの正しさを自分で判断できるようになること
- エラーの原因を自分で考えられるようになること
- 将来的にエンジニアとして活動できる基礎力を身につけること
- AIに依存するのではなく、AIを**道具として使いこなせる**人間になること

## コミットルール

### タイミング

- 一つの作業が完了するたびにコミットする
- 「全部できてからまとめてコミット」はしない

### メッセージの書き方（gitmoji）

#### よく使う基本セット

- 🎉 `:tada:` 初回コミット
- ✨ `:sparkles:` 新機能追加
- ⚡️ `:zap:` 既存機能に機能を追加
- 🔥 `:fire:` コードやファイルの削除
- 🐛 `:bug:` バグ修正（機能そのものに問題がある場合）
- 🩹 `:adhesive_bandage:` ちょっとした修正（一瞬で直せる軽微なもの）
- 🚧 `:construction:` 作業中（まだ動かない）

#### その他

- ♻️ `:recycle:` リファクタリング
- 🚚 `:truck:` ファイル移動・ファイル名変更
- 🔧 `:wrench:` 構成変更や設定ファイルの修正
- 📝 `:memo:` コメントをコード内に記載
- ⚰️ `:coffin:` 不要なコードの削除
- 💬 `:speech_balloon:` 一時的にコメントアウト

### 例

- 🎉 初回コミット
- ✨ トレード履歴取得APIを追加
- 🔧 .gitignoreを追加
- 🐛 損益計算のバグを修正
- 🩹 typoを修正
