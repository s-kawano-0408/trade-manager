## 1.5 MT4連携の仕組み

### 全体の流れ

```
┌──────────────────┐
│ MT4（XM口座）     │
│                  │
│ EA（自動プログラム）│
│ を常駐させる      │
└────────┬─────────┘
         │ トレード決済のたびに
         │ HTTP POST（JSON）
         ↓
┌──────────────────┐
│ ngrok             │  ← ローカルPCのFastAPIを
│ （トンネルツール）  │    インターネットに一時公開
└────────┬─────────┘
         │
         ↓
┌──────────────────┐
│ FastAPI（ローカル） │
│ localhost:8000    │
└────────┬─────────┘
         │ 保存
         ↓
┌──────────────────┐
│ Supabase（DB）    │
└──────────────────┘
```

### EA（Expert Advisor）について

- MQL4というMT4専用の言語で書く小さなプログラム
- MT4のチャートに貼り付けて常駐させる
- トレードが決済されたタイミングを検知して、FastAPIにデータを送る
- EA内にngrokのURL + APIキーを設定する

### EA → FastAPI 送信データ（JSON）

```json
{
  "ticket": 12345678,
  "symbol": "XAUUSD",
  "type": 0,
  "lots": 0.10,
  "open_price": 2045.50,
  "close_price": 2046.80,
  "open_time": "2026-03-01 09:15:00",
  "close_time": "2026-03-01 09:18:30",
  "profit": 1300,
  "commission": -100,
  "swap": 0,
  "magic_number": 0,
  "comment": ""
}
```

### XM口座側の設定

1. MT4の「ツール」→「オプション」→「エキスパートアドバイザ」タブ
2. 「自動売買を許可する」にチェック
3. 「DLLの使用を許可する」にチェック
4. 「WebRequestを許可するURLリスト」にngrokのURLを追加

### ngrok について

- ローカルPC（localhost:8000）をインターネットに一時的に公開するツール
- PCを起動している間だけ有効（PC閉じたらMT4から届かなくなる）
- お試し段階で使用し、本番はサーバーデプロイ（Render等）に移行する
- 無料プランでは毎回URLが変わるため、EA側のURL書き換えが必要

### セキュリティ（認証）

```
MT4 EA → FastAPI への通信:
  ヘッダーに X-API-Key を付与
  FastAPI側で .env のキーと一致するか確認
  一致しなければ拒否（401 Unauthorized）
```

## 6. データフロー

### 6.1 トレードデータの流れ

```
MT4でトレード決済
      ↓
EA（MQL4）がトレード情報をJSON化
      ↓
HTTP POST → FastAPI /api/trades/webhook
      ↓
FastAPIがバリデーション + 重複チェック（ticket番号）
      ↓
Supabase（PostgreSQL）に保存
      ↓
フロントエンドが GET /api/trades で取得して表示
```

### 6.2 AI分析の流れ

```
ユーザーが「分析を実行」ボタンを押す
      ↓
フロントエンド → POST /api/analysis（期間を指定）
      ↓
FastAPIが指定期間のトレードデータをDBから取得
      ↓
トレードデータ + 日記データを整形してプロンプトに組み込む
      ↓
Claude API に送信
      ↓
分析結果（テキスト）をフロントエンドに返却
      ↓
画面に表示
```

## 7. 技術選定の理由

| 技術 | 選定理由 |
|------|----------|
| React | コンポーネント指向でUI構築しやすい。学習リソースが豊富 |
| Vite | ビルドが高速。React公式推奨のビルドツール |
| TypeScript | 型があることでバグを未然に防げる。エディタの補完が効く |
| TailwindCSS | クラス名だけでスタイリング可能。CSSを別ファイルで管理する手間がない |
| shadcn/ui | 高品質なUIコンポーネント。必要なものだけ追加できる |
| Recharts | Reactと相性の良いグラフライブラリ |
| FastAPI | Pythonで高速なAPI構築が可能。自動ドキュメント生成 |
| SQLAlchemy | PythonのDB操作をオブジェクト指向で書ける |
| Supabase | PostgreSQLのホスティング。無料枠あり |
| Claude API | 日本語での自然な分析コメント生成が得意 |
