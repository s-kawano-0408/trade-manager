## 1.5 トレードデータの入力方法

### 背景

Mac版MT4はEA（自動プログラム）でのWebhook送信が現実的でないため、以下の2つの方法でデータを入力する。

### ① 手入力

- 画面のフォームからトレードを1件ずつ入力
- 入力項目：決済日時・売買方向・Lot・エントリー価格・決済価格・損益・手数料・スワップ
- `ticket` 番号はシステムが自動生成（タイムスタンプベース）

### ② MT4 HTMLインポート

MT4の画面から口座履歴をHTMLで書き出して、まとめて取り込む方法。

```
MT4「ターミナル」→「口座履歴」→ 右クリック →「レポートを保存」
      ↓ HTMLファイルを書き出す
画面の「HTMLインポート」ボタンからファイルを選択
      ↓
FastAPI がHTMLをパース
      ↓
SQLite に保存
```

### HTMLインポート時の上書きルール

HTMLに含まれる日付を基準に、その日のデータをまるごと置き換える。

- HTMLに「2026/03/07」のデータが10件 → 同日の手入力データを削除して10件で上書き
- HTMLに含まれない日付のデータはそのまま残す

→ **HTMLが常に正の情報とみなす**


### 3.2 API設計

#### トレード関連

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/trades` | トレード一覧を取得 |
| POST | `/api/trades/manual` | 手入力でトレードを1件登録 |
| POST | `/api/trades/import` | MT4のHTMLファイルを一括インポート（日付単位で上書き） |
| PUT | `/api/trades/{id}` | トレードを更新 |
| DELETE | `/api/trades/{id}` | トレードを削除 |
| GET | `/api/trades/summary` | サマリー（勝率・損益等）を取得 |

#### 日記関連

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/diary` | 日記一覧を取得 |
| POST | `/api/diary` | 日記を新規作成 |
| PUT | `/api/diary/{id}` | 日記を更新 |
| DELETE | `/api/diary/{id}` | 日記を削除 |

#### HTMLインポートの上書きルール

```
POST /api/trades/import にHTMLファイルを送信
  ↓
HTMLに含まれる日付を一覧取得
  ↓
その日付に一致する既存レコードを全削除
  ↓
HTMLのデータをINSERT
  ↓
HTMLに含まれない日付のデータはそのまま
```

## 6. 技術選定の理由

| 技術 | 選定理由 |
|------|----------|
| React | コンポーネント指向でUI構築しやすい。学習リソースが豊富 |
| Vite | ビルドが高速。React公式推奨のビルドツール |
| TypeScript | 型があることでバグを未然に防げる。エディタの補完が効く |
| TailwindCSS | クラス名だけでスタイリング可能。CSSを別ファイルで管理する手間がない |
| shadcn/ui | 高品質なUIコンポーネント。必要なものだけ追加できる |
| Recharts | Reactと相性の良いグラフライブラリ |
| FastAPI | Pythonで高速なAPI構築が可能。自動ドキュメント生成 |
| SQLAlchemy | PythonのDB操作をオブジェクト指向で書ける |
| SQLite | ローカル動作のみのため、インストール不要・設定ゼロのSQLiteを採用 |
