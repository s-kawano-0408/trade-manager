# 要件定義書 - Trade Manager

## 1. プロジェクト概要

### 1.1 目的
ゴールド（XAU/USD）スキャルピングトレーダーの収支を管理・分析するためのWebアプリケーション。
MT4（XM口座）のトレードデータを自動取得し、損益の可視化・心理分析・リスク管理を行う。

### 1.2 対象ユーザー
- 個人トレーダー（自分自身）
- ゴールドスキャルピングを主な手法とする
- 1日あたり数回〜数十回のトレードを行う

### 1.3 解決する課題
- トレード結果を手動で記録する手間をなくす
- 感情的なトレード（ポジポジ病・リベンジトレード）を数値で検出する
- 「なんとなく負けた」ではなく、データに基づいた振り返りを可能にする

## 2. 機能要件

### 2.1 ダッシュボード画面

| 項目 | 内容 |
|------|------|
| 目的 | 本日・週次・月次の損益を一目で把握する |
| 表示内容 | 日次損益グラフ（棒グラフ/折れ線）、累計損益、勝率、PF（プロフィットファクター）、平均損益、最大ドローダウン、本日の取引回数 |
| 期間切替 | 本日 / 今週 / 今月 / カスタム期間 |
| 自動更新 | MT4からデータが送られたらリアルタイムで反映 |

### 2.2 トレード一覧画面

| 項目 | 内容 |
|------|------|
| 目的 | 全トレード履歴を確認・検索する |
| 表示項目 | チケット番号、通貨ペア、売買方向（BUY/SELL）、ロットサイズ、エントリー価格、決済価格、損益（pips）、損益（円）、エントリー時刻、決済時刻、保有時間 |
| フィルタ | 期間、売買方向、勝ち/負け、ロットサイズ範囲 |
| ソート | 各カラムでソート可能 |
| ページネーション | 1ページあたり20〜50件を表示 |

### 2.3 AI分析画面

| 項目 | 内容 |
|------|------|
| 目的 | トレードの心理的傾向・リスク管理状況を分析する |
| 起動方法 | ユーザーが「分析」ボタンを押下したときのみ実行（API節約のため自動実行しない） |
| 分析対象期間 | 本日 / 今週 / 今月 / カスタム |
| 使用API | Claude API（Anthropic） |

#### 分析内容

**ポジポジ病の検出**
- 1日のトレード回数が直近平均の2倍以上 → 警告
- 短時間（5分以内）に連続エントリー → 警告

**ロット異常の検出**
- 直近平均ロットの2倍以上のトレード → 警告
- 連敗直後のロット増加 → 警告

**リベンジトレードの検出**
- 大きな損失（直近平均損失の2倍以上）の直後（10分以内）にエントリー → 警告

**時間帯分析**
- 時間帯別の勝率・損益を算出
- 成績が悪い時間帯に集中してトレードしていないか分析

**総合コメント**
- 上記の分析結果をもとに、日本語の自然なコメントを生成

### 2.4 経済指標カレンダー画面

| 項目 | 内容 |
|------|------|
| 目的 | ゴールドに影響する経済指標の日程を把握する |
| 表示項目 | 日付、時刻（日本時間）、指標名、重要度（高/中/低）、前回値、予想値、結果値 |
| 重要指標 | NFP（米雇用統計）、FOMC、CPI（消費者物価指数）、GDP、小売売上高、ISM製造業指数、Powell議長発言 |
| データ取得 | 外部APIから自動取得（候補：Forex Factory、Investing.com API等） |
| フィルタ | 重要度、期間 |

### 2.5 トレード日記画面

| 項目 | 内容 |
|------|------|
| 目的 | 日々のトレードの振り返りを記録する |
| 入力項目 | 日付、その日の相場の感想（テキスト自由記述）、メンタル状態（良い/普通/悪い）、反省点・気づき、翌日への意識 |
| 機能 | 新規作成、編集、削除、一覧表示 |
| 補助表示 | その日のトレード成績サマリー（勝率・損益）を自動で横に表示 |
| AI連携 | AI分析時に日記の内容も考慮して分析（メンタル状態と成績の相関） |

## 3. 非機能要件

### 3.1 パフォーマンス
- ダッシュボードの初回表示：3秒以内
- トレードデータのWebhook受信から画面反映：5秒以内

### 3.2 セキュリティ
- APIキー・DB接続情報はサーバーサイドの環境変数で管理
- フロントエンドからDBへの直接アクセスは禁止（必ずAPI経由）
- Supabaseの接続URLは外部に漏らさない

### 3.3 可用性
- 個人利用のため高可用性は不要
- ローカル開発環境 + Supabase（クラウドDB）構成

### 3.4 対応ブラウザ
- Chrome（最新版）を主対象
- レスポンシブ対応（PC優先、スマホは閲覧可能レベル）

## 4. データ要件

### 4.1 MT4から受信するトレードデータ

| フィールド | 型 | 説明 |
|-----------|-----|------|
| ticket | integer | チケット番号（MT4で一意） |
| symbol | string | 通貨ペア（例：XAUUSD） |
| type | integer | 売買方向（0=BUY, 1=SELL） |
| lots | float | ロットサイズ |
| open_price | float | エントリー価格 |
| close_price | float | 決済価格 |
| open_time | datetime | エントリー時刻 |
| close_time | datetime | 決済時刻 |
| profit | float | 損益（口座通貨） |
| commission | float | 手数料 |
| swap | float | スワップ |
| magic_number | integer | EA識別番号 |
| comment | string | コメント |

### 4.2 トレード日記データ

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | integer | 一意のID（自動採番） |
| date | date | 日付 |
| market_comment | text | 相場の感想 |
| mental_state | string | メンタル状態（good/normal/bad） |
| reflection | text | 反省点・気づき |
| next_day_plan | text | 翌日への意識 |
| created_at | datetime | 作成日時 |
| updated_at | datetime | 更新日時 |

## 5. 外部システム連携

### 5.1 MT4（MQL4 EA）→ FastAPI
- 通信方式：HTTP POST（Webhook）
- タイミング：トレード決済時に自動送信
- データ形式：JSON
- セキュリティ：APIキーによる認証

### 5.2 FastAPI → Supabase（PostgreSQL）
- 通信方式：SQLAlchemy ORM経由
- 接続情報：環境変数（.env）で管理

### 5.3 フロントエンド → Claude API
- 通信方式：バックエンド経由でAPI呼び出し（フロントから直接呼ばない）
- タイミング：ユーザーが分析ボタンを押下したときのみ
- コスト管理：Proプランの上限を考慮し、オンデマンド実行のみ
