# 要件定義書 - Trade Manager

## 1. プロジェクト概要

### 1.1 目的
ゴールド（XAU/USD）スキャルピングトレーダーの収支を管理・分析するためのWebアプリケーション。
MT4（XM口座）のトレードデータを取り込み、損益の可視化・振り返りを行う。

### 1.2 対象ユーザー
- 個人トレーダー（自分自身）
- ゴールドスキャルピングを主な手法とする
- 1日あたり数回〜数十回のトレードを行う

### 1.3 解決する課題
- MT4のHTMLエクスポートで手軽にトレードデータを取り込める
- 手入力でも1件ずつ登録できる（HTMLがないときの補完）
- HTMLインポート時は日付単位で上書きするため、手入力との整合性が自動で保たれる
- 「なんとなく負けた」ではなく、データに基づいた振り返りを可能にする

## 2. 機能要件

### 2.1 ダッシュボード画面

| 項目 | 内容 |
|------|------|
| 目的 | 損益を一目で把握する |
| 表示内容 | 合計損益、勝率、総取引数、日次損益グラフ（棒グラフ）、直近トレード5件 |

### 2.2 トレード一覧画面

| 項目 | 内容 |
|------|------|
| 目的 | 全トレード履歴を確認・管理する |
| 表示項目 | 売買方向（BUY/SELL）、ロットサイズ、エントリー価格、決済価格、損益（円）、決済日時、種別（手入力/インポート） |
| データ入力 | ① 手入力フォーム（1件ずつ）、② MT4 HTMLインポート（一括） |
| HTML上書きルール | HTMLに含まれる日付のデータは、その日の手入力データを削除してHTMLで上書き |
| 編集・削除 | 各トレードを編集・削除できる |

### 2.3 トレード日記画面

| 項目 | 内容 |
|------|------|
| 目的 | 日々のトレードの振り返りを記録する |
| 入力項目 | 日付、その日の相場の感想（テキスト自由記述）、メンタル状態（良い/普通/悪い）、反省点・気づき、翌日への意識 |
| 機能 | 新規作成、編集、削除、一覧表示 |

## 3. 非機能要件

### 3.1 動作環境
- **ローカル動作のみ**（クラウドへのデプロイは行わない）
- フロントエンド：`http://localhost:5173`（Vite開発サーバー）
- バックエンド：`http://localhost:8000`（uvicorn）

### 3.2 セキュリティ
- ローカル動作のみのため、外部公開はしない
- APIキー等の機密情報は `.env` で管理し、Gitには含めない

### 3.3 対応ブラウザ
- Chrome（最新版）を主対象

## 4. データ要件

### 4.1 トレードデータ

| フィールド | 型 | 説明 |
|-----------|-----|------|
| ticket | integer | チケット番号（手入力時はタイムスタンプから自動生成） |
| symbol | string | 通貨ペア（例：GOLD） |
| type | integer | 売買方向（0=BUY, 1=SELL） |
| lots | float | ロットサイズ |
| open_price | float | エントリー価格 |
| close_price | float | 決済価格 |
| open_time | datetime | エントリー時刻 |
| close_time | datetime | 決済時刻 |
| profit | float | 損益（口座通貨） |
| commission | float | 手数料 |
| swap | float | スワップ |
| magic_number | integer | EA識別番号（手入力時は0） |
| comment | string | コメント |
| source | string | 入力方法（"manual" or "import"） |

### 4.2 トレード日記データ

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | integer | 一意のID（自動採番） |
| date | date | 日付 |
| market_comment | text | 相場の感想 |
| mental_state | string | メンタル状態（good/normal/bad） |
| reflection | text | 反省点・気づき |
| next_day_plan | text | 翌日への意識 |
| created_at | datetime | 作成日時 |
| updated_at | datetime | 更新日時 |

## 5. 外部システム連携

### 5.1 トレードデータ入力方法

**① 手入力**
- 通信方式：画面フォーム → HTTP POST `/api/trades/manual`
- タイミング：ユーザーが手動で入力したとき

**② HTMLインポート**
- 通信方式：ファイルアップロード → HTTP POST `/api/trades/import`
- タイミング：ユーザーがHTMLファイルを選択したとき
- 元データ：MT4「ターミナル」→「口座履歴」→「レポートを保存」で書き出したHTML

### 5.2 データベース
- **SQLite**（ローカルファイルDB）を使用
- ファイルは `backend/trade_manager.db` に自動生成される
- アプリ起動時にテーブルを自動作成（事前のDB準備不要）
